Data Flow
every page load
1
Visitor loads index.html — check localStorage for cached weather
Cache is valid if: version matches v4, date matches today, age < 30 min
2
Cache miss → call /.netlify/functions/get-daily
Server checks Supabase for today's row (keyed by NYC date YYYY-MM-DD)
3a
Row exists + under 3 hours old → return cached row, skip OWM call
3b
Row exists + over 3 hours old → fetch fresh weather from OpenWeatherMap, PATCH weather fields only (score never changes here)
3c
No row yet → fetch OWM, run scoring algorithm, INSERT full row into Supabase
Score is set once here and never touched by refresh again
4
After returning the row: if past 9AM NYC and no synopsis_approved exists, auto-generate synopsis via Claude Haiku and save it
5
Client renders: score, headline, penalty pills, outfit tags, 5-day forecast, synopsis
Supabase score always wins over client-side score. Synopsis only shows if approved = true.
6
track-visit fires in background — logs the visit for analytics, no UI impact

Score Rules
Set once — score is calculated and saved when the daily row is first created. Never recalculated by a refresh.
Force Refresh — updates all weather data (temp, high, rain, etc.) from OWM but does not touch the score.
Score Override — the only way to change the score after row creation. Admin panel → weather section → type a number and click Set Score. Writes directly to Supabase.
Client vs server — both pages run scoreWeather() client-side as a fallback, but weather.score from Supabase always wins if it exists.
Synopsis System
Auto-Generation (server-side)
Triggers inside get-daily.js after returning the row — if past 9AM NYC and no synopsis_approved on today's row
Pulls up to 8 writing examples from Supabase: recent approved daily rows take priority, then seeded synopsis_examples table entries fill remaining slots
Sends to Claude Haiku with weather data + examples. Prompt requires: 1–2 sentences, NYC energy, lowercase, no hashtags, one emoji max
Saves result directly to synopsis_approved + sets approved = true — live immediately
Manual (Admin Panel)
Click ↻ Regenerate → calls generate-synopsis.js (same Claude logic) → lands in textarea as a draft, not yet live
Click ✓ Approve & Publish → calls approve-synopsis.js → PATCHes Supabase → live to all visitors immediately
Editing the textarea after approving shows ↑ Override & Publish — you have to re-approve to push the edit live
States
not generated
No draft or approved text. Click Regenerate.
draft — not yet published
AI wrote it, not approved. Visitors see "today's vibe dropping soon."
edited — not yet published
You edited the textarea after approval. Hit Override & Publish to go live.
approved & live
In Supabase, visible to all visitors. Source of truth.
Caching
Client cache — stored in localStorage under key cnw_weather. TTL: 30 minutes. Busted if date changes, version mismatch, or TTL expired.
Cache version — currently v4. Must match in both index.html and admin.html. Bump both to force all devices to re-fetch.
Server cache — Supabase row is the cache. Stale threshold: 3 hours. Exceeded → fresh OWM fetch + PATCH to Supabase.
Force Refresh (admin only) — bypasses 3hr threshold, fetches OWM immediately, writes to Supabase, busts local client cache.
Outfit Logic
Daytime outfit uses daily HIGH. Night outfit uses daily LOW.
Night outfit row only shows if temp swing (HIGH − LOW) is 15°F or more and LOW is not null.

Netlify Functions

Supabase Tables
daily
One row per day. Key: date_key (NYC date, YYYY-MM-DD)
Weather fields: temp high low feels_like condition humidity precip_chance wind_speed forecast (JSON)
Score fields: score penalties (array)
Synopsis fields: synopsis_draft synopsis_approved approved (bool)
Meta: updated_at
synopsis_examples
Manually seeded writing examples used to train Claude's voice. Fields: synopsis temp feels_like condition precip_chance score created_at
Recent approved daily rows take priority over these as examples. These fill remaining slots up to 8 total.
subscribers
Phone numbers for SMS beta. Fields: phone created_at active (bool)
